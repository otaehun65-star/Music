# 🎵 K-Vibe (한국 음악 추천 및 공유 플랫폼)

**공식 접속 주소 (Main Address):** [https://otaehun65-star.github.io/Music/](https://otaehun65-star.github.io/Music/)

---

## 📝 프로젝트 소개
우리나라(한국)의 다양한 음악을 포함하여, 이용자들이 자신이 좋아하는 음악을 소개하고 서로의 음악 추천을 공유할 수 있는 커뮤니티 성격의 웹 플랫폼입니다.
현재 **GitHub Pages**를 통해 전 세계 어디서든 접속 가능한 라이브 상태입니다.

## 🌟 주요 기능 (Key Features)

1. **회원 시스템 (User Management)**
   - 회원가입 및 로그인 (이메일 및 소셜 연동)
   - 개인 프로필 및 취향 태그 설정

2. **음악 추천 글 작성 (Music Posting)**
   - 음악 정보 입력: 곡명, 아티스트명, 장르 태그
   - 음악 감상평 및 리뷰 작성
   - 스트리밍 연동: YouTube, Spotify 등 바로듣기 링크 첨부 가능

3. **피드 및 탐색 (Feed & Explore)**
   - 메인 피드: 최신, 인기 추천곡 타임라인
   - 장르별 필터링 (K-Pop, 인디, 힙합, 발라드 등)
   - 곡명, 가수를 통한 통합 검색 기능

4. **상호작용 및 소셜 (Interaction)**
   - 게시글 '좋아요' 및 '댓글' 기능
   - 마음에 드는 추천곡을 내 웹 플레이리스트(보관함)에 담기
   - 다른 유저 팔로우 기능

## 🛠️ 작업 방법 (Work Plan)

### 1단계: 기획 및 화면 설계 (Week 1)
- 핵심 유저 스토리 및 요구사항 정의
- Figma 등을 활용한 반응형 UI/UX 화면(와이어프레임) 스케치
- 데이터베이스 스키마 모델링 추가 설계

### 2단계: 프론트엔드 개발 (Week 2-3)
- **기술 스택**: React (비트(Vite) 또는 Next.js), TailwindCSS (또는 Styled-Components)
- 기본 컴포넌트(버튼, 폼, 카드) 구현
- 추천 리스트, 상세 페이지, 작성 폼 등 화면 라우팅 처리 연동
- 백엔드 REST API 구현 전 MOCK 데이터를 활용한 테스트

### 3단계: 백엔드 및 데이터베이스 구축 (Week 2-3)
- **기술 스택**: Node.js(Express) 또는 Spring Boot
- **데이터베이스**: MySQL 또는 PostgreSQL
- 사용자 인증(JWT/세션) 및 스토리지(프로필 등 이미지) 로직 구현
- 주요 API 로직 개발 (게시글 CRUD, 좋아요, 댓글)

### 4단계: 연동 및 테스트 (Week 4)
- 프론트-백엔드 API 연동 작업
- 버그 픽스 및 전체적인 이용 흐름 (가입 -> 게시글 작성 -> 조회) 확인

### 5단계: 배포 (Week 5)
- AWS, Vercel 등을 통한 호스팅 및 배포, 도메인 연결
